## 1. The Core Concept: DNA & The Plant
We define the software lifecycle through a biological metaphor:

*   **The Seed (Universal Kernel)**: A master protocol containing the fundamental "genetic code" of high-quality engineering (Git flows, Atomic Design, Safety Checks). It is project-agnostic.
*   **The Plant (Project Manifest)**: When the Seed is planted in a specific project (e.g., specific customer needs), it grows into a **Project Manifest**. It inherits the Seed's traits but develops new features (leaves/branches) to survive its specific environment.
*   **The Harvest (Evolutionary Loop)**: At the end of the project, we do not just ship the code. We extract the "strongest traits" developed by the Plant (e.g., a new way to handle errors) and merge them back into the Seed. The next Seed is now genetically superior.

## 2. Methodology: Dual-Layer Architecture

### Layer 1: The Universal Prompt Core (The Genotype)
*   **Role**: The immutable foundation.
*   **Content**: "Always start with analysis", "Never use 'any' type", "Commit often".
*   **Behavior**: Read-Only for the AI during a project.

### Layer 2: The Project Manifest (The Phenotype)
*   **Role**: The adaptive layer.
*   **Content**: "Use Recharts", "Customer hates blue buttons", "Legacy API workarounds".
*   **Behavior**: Read-Write. Continuously updated via **Validated Task Sequences**.
